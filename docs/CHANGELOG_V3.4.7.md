# CHANGELOG v3.4.7

## 새 레이팅 산식

- 초기 승률은 모두 0.5, 초기 KDA는 모두 3.0으로 동일하다고 보고 초기 산식에서 실질적으로 제외
- 티어 점수 30%, 라인 점수 15%, 챔피언 폭 15%, 상위 10% 보너스로 초기 파워 점수 산출
- z-score 표준편차가 0이면 해당 항목을 0으로 처리
- 초기 파워 점수는 평균 1500, 표준편차 200 스케일
- 기존 경기를 `played_at`, `match_id` 순으로 다시 재생
- TOP/JUNGLE/MID/ADC/SUPPORT 맞라이너끼리 레이팅을 비교
- 라인전 승패를 별도로 입력하지 않고 최종 팀 승패를 동일 포지션 맞대결 Elo에 적용
- K 값 32

## 기존 데이터

- 기존 경기 삭제 및 재입력 불필요
- 멤버 화면의 `레이팅 전체 재계산` 버튼으로 현재 저장된 모든 경기를 시간순 재반영
- 경기 신규 등록 및 수정 시 전체 레이팅 자동 재계산

## 팀 자동 배정

- 10명 중 5명 팀 조합 126개 검사
- 각 팀에서 5개 라인 배치 120개 검사
- 라인 적합도 합을 높이고 양 팀 레이팅 차이를 줄이는 조합 선택
- 자동 배정 결과에 `assignedPosition` 포함

## 현재 데이터 해석

- 라인 능력치는 `member_position` 우선순위로 계산
  - 1순위: skill 2
  - 2~3순위 또는 FILL: skill 1
- 챔피언 폭은 기존 `inhouse_match_player`에서 포지션별 고유 챔피언 수로 계산
