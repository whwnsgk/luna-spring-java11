# CHANGELOG v3.5.4

## Java heap OutOfMemoryError 근본 수정

스택트레이스의 실제 원인은 `TeamController.assigned()`에서 후보 한 개마다
10명의 `LinkedHashMap`을 새로 만들던 구조였습니다.

강제 탐색의 최대 후보 수는 약 181만 개이므로, 상위 100개만 큐에 남기더라도
평가 과정에서 수천만 개의 Map 객체가 생성되어 힙이 고갈될 수 있었습니다.

변경 후:

- 후보 평가 중에는 멤버 Map 복사 없음
- 원본 멤버 목록과 작은 라인 인덱스 배열만 사용
- 최종 상위 후보를 응답할 때만 10명의 배정 Map 생성
- 상위 100개 우선순위 큐 유지

## 경매 메모리 정리

- 완료 후 30분이 지나고 접속자가 없는 경매방 자동 삭제
- 경매방이 50개를 초과하면 접속자가 없는 오래된 방부터 정리
- 새 경매방 생성 전에도 정리 수행

## 중요

`OutOfMemoryError`가 한 번 발생한 JVM은 내부 상태가 불안정할 수 있으므로
새 버전 적용 후 반드시 서버 프로세스를 완전히 종료하고 다시 시작해야 합니다.
