# CHANGELOG v3.4.13

## 레이팅 600 고정 문제 수정

포지션 과밀 보정 후 최소값을 600으로 강제하던 코드를 제거했습니다.

```
기존: max(600, 원래 점수 - 페널티)
변경: max(0, 원래 점수 - 페널티)
```

자동 팀 짜기와 경매 모두 실제 유효 레이팅 차이를 유지합니다.

## 자동 팀 짜기 시네마틱

자동 팀 생성 성공 시 약 3초간 화면 전체 시네마틱을 표시합니다.

- 영상: `/media/team-balance-video.mp4`
- 문구: `완벽한 밸런스!`
- 영상이 없으면 텍스트 애니메이션만 표시
- `/media/event-video.mp4`를 공통 대체 영상으로 사용 가능

## 경매 낙찰 시네마틱

후보의 레이팅 비중과 전체 경매 예산으로 적정가를 계산합니다.

```
적정가 = 양 팀 총 기본 토큰 × 후보 레이팅 / 경매 후보 8명의 레이팅 합
```

- 높은 레이팅 후보를 적정가의 80% 이하에 낙찰:
  - `완벽한 입찰!`
  - `/media/perfect-bid-video.mp4`
- 적정가의 125% 이상에 낙찰:
  - `너무 비싼 입찰!`
  - `/media/expensive-bid-video.mp4`
- 그 사이의 일반 입찰은 기존 SOLD 연출만 표시

## 경매 종료 승률

경매 완료 화면과 전체 화면 종료 연출에 양 팀 예상 승률을 크게 표시합니다.

```
BLUE 예상 승률 = 1 / (1 + 10^((RED 총점 - BLUE 총점) / 400))
```
